# Prototype-Diff Decisions Log

Companion to `docs/PROTOTYPE_DIFF_AUDIT.md`. Records each decision the user
makes while walking through the audit, plus the running list of items to
submit back to Claude Design for clarification or design-side completion.

**Started:** 2026-05-28
**Guiding principle:** Visually, the prototype takes precedence (the user
wants the UI/UX to look EXACTLY like the prototype). Functionally, the
current app takes precedence — no functional regressions.

**Architectural constraint (set 2026-05-28 at decision 40):**
> **No remote storage of any kind.** This is a hard constraint across the
> entire app, applied retroactively to every prior decision in this log.
> Every prototype feature that implied a server, hosted URL, verified-link
> system, read-receipt, share-history backend, hosted-QR endpoint, or
> over-the-wire token exchange must be re-spec'd as local-only — typically
> via QR-scan handoff between devices, system mail/SMS composers, or signed
> local files that the user transmits manually. Items #17 (Share Sheet +
> Wallet QR), #24 (Past directive share history), #26 (Revocation
> notifications), and the Agent Acceptance Receipt all inherit this
> constraint. The privacy-policy URL (V4-C1) is the only HTTP endpoint
> we'll host — it's a static page for Play/App Store compliance, not app
> data.
